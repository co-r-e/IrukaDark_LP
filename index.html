What does this mean?

    FAQ

    Answers to common questions about IrukaDark

    What platforms are supported?

    macOS. Download the installer from the Downloads page or GitHub Releases.

    How is it different from a translator?

    It explains text in context—beyond word‑for‑word translation—so you get intent, background, and plain‑language explanations instantly.

    Does it store my data?

    No. IrukaDark stores no data and does not train on your content, helping protect your privacy.

    Do I need a GitHub account or developer tools?

    No for normal use — just download and install. A GitHub account, Git, and Node.js are only needed if you want to build from source.

    How do I use it quickly?

    Select text and press Option+A for an explanation. For images and graphs, select an area and press Option+S for screenshot analysis.

    Is it free and open source?

    Yes. IrukaDark is open source and free to use. Download prebuilt installers from GitHub Releases, or build from source if you prefer.

    Does it require an internet connection?

    Yes. IrukaDark connects to AI services to generate explanations. Only the content you select (or capture) is sent over HTTPS; IrukaDark does not store your data.

    What OS permissions are required?

    Permission macOS
    Screen capture Required: Screen Recording
    Accessibility Optional for global shortcuts / in‑app selection
    Admin rights Not required
    Network (HTTPS) Allow outbound HTTPS
    Global shortcuts System Settings → Keyboard/Shortcuts

    Item macOS

    Build from source (optional) Requires Git and Node.js LTS if you choose to build yourself

    Related Terms & Commands

    Overview

    A compact cheat sheet of must‑know terms and commands to set up and maintain IrukaDark.

    File/Folder Basics (Terminal)

    • cd dir — Change directory (e.g., cd irukadark_lp).
    • ls — List contents.
    • pwd — Print current directory path.
    • mkdir name — Create a new folder.
    • cp src dest — Copy file/folder.
    • mv src dest — Move/rename file/folder.

    Git (Get/Update Latest)

    • git clone <URL> — Get the repo the first time.
    • git pull — Update to the latest in an existing folder.
    • git status — Check changes.

    Node.js / npm

    • node -v / npm -v — Check versions (prefer LTS).
    • npm install — Install dependencies (required).

    Terms

    • Repository — Project storage managed by Git.
    • Clone — Copy the repository to your machine.
    • Pull — Fetch and merge latest remote changes.
    • PATH — Env var that finds executables.
    • Terminal/Shell — Where you run commands (bash, zsh).

    Note: Destructive commands like rm -rf are risky. Always confirm your current directory with pwd before running them.
